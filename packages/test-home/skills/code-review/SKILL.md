# 代码审查 Skill

## 审查要点

1. **代码质量**：变量命名、函数长度、代码重复、复杂度
2. **性能**：循环优化、内存泄漏、不必要的计算
3. **安全**：SQL 注入、XSS、权限检查、输入验证
4. **可维护性**：注释、文档、测试覆盖、模块化

## 工作流

1. 先了解代码的整体结构和目的
2. 逐行检查关键逻辑
3. 用 git diff 查看改动
4. 给出具体、可操作的改进建议

## 输出格式

```
## 审查结果

### 问题 1: [严重程度] 问题描述
- 位置: 文件:行号
- 建议: 改进方案

### 问题 2: ...
```
